## Architecture

This is a Flutter app for a smart irrigation system with AI plant monitoring. It uses **Firebase Realtime Database** as the single source of truth — both for reading sensor data (streams) and for writing commands that a hardware controller picks up.

### Data flow

```
Hardware/Edge device
  → writes to Firebase RTDB
    → Flutter app reads via streams
      → user commands written back to Firebase
        → hardware reads and executes
```

### Firebase RTDB structure

| Path | Direction | Description |
|------|-----------|-------------|
| `sensors/soil_moisture` | read | % moisture |
| `sensors/temperature` | read | °C |
| `sensors/water_level` | read | % level |
| `sensors/current` | read | Amps draw |
| `status/system_state` | read | `NORMAL` / `WARNING` / `FAULT` |
| `status/pump` | read | `ON` / `OFF` |
| `status/gantry_x`, `gantry_y` | read | gantry position |
| `ai/latest_detection` | read | `{class, confidence, image_url}` |
| `ai/active_protocol` | read | `{status, zone_x, zone_y, grace_remaining}` |
| `alerts/history` | read | list of `{severity, message, timestamp}` |
| `commands/pump` | **write** | `{state, timestamp, source}` |
| `commands/emergency_stop` | **write** | `true` |
| `commands/gantry_move` | **write** | `{x, y, timestamp}` |
| `commands/cancel_ai_protocol` | **write** | `true` |

### Key files

- `lib/services/firebase_service.dart` — all read streams as static getters; the single place to add new sensor/state streams
- `lib/services/stt_service.dart` — singleton Speech-to-Text wrapper; initialized once in `main()`, exposes `startListening`/`stopListening`
- `lib/theme/app_theme.dart` — Material 3 theme with a green color palette (greenDeep → greenPale) and DM Serif Display / DM Sans fonts
- `lib/main.dart` — app entry, Firebase init, STT init, bottom nav (`MainNavigation`) with 5 tabs

### Screens

| Tab | Screen | Purpose |
|-----|--------|---------|
| Home | `HomeScreen` | Static landing page — hero, quick stats, "how it works" cards |
| Dashboard | `DashboardScreen` | Live sensor grid + AI detection card (`FirebaseService` streams) |
| Controls | `ActuatorScreen` | Pump toggle, emergency stop, gantry test move (direct DB writes) |
| AI Monitor | `AiMonitorScreen` | Latest detection image + active irrigation protocol |
| Alerts | `AlertsScreen` | Alert history sorted by timestamp |

`SttTestScreen` is a standalone debug screen pushed from the FAB mic button.

### Widgets

- `SensorCard` — takes a `Stream<double>` and renders live value with unit/icon
- `DetectionCard` — streams `FirebaseService.latestDetection`, shows class + confidence bar
- `StatusBadge` — streams `FirebaseService.systemState`, renders colored pill (NORMAL/WARNING/FAULT)

### AI detection classes

`Healthy`, `Early_Blight`, `Late_Blight`, `Pest`, `Nutrient_Deficiency` — each has a distinct color in `DetectionCard._colorForClass`.

## Setup notes

- `lib/firebase_options.dart` is generated by `flutterfire configure` and is gitignored; it must exist for the app to build.
- Microphone permission is requested at startup via `SttService.initialize()` — on Android this requires the `RECORD_AUDIO` permission in the manifest.
